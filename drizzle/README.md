# Migrations

This folder is managed by [drizzle-kit](https://orm.drizzle.team/kit-docs/overview).

## Status

`0000_messy_proteus.sql` was generated by `drizzle-kit generate`.
`0001_add_auth.sql` through `0005_friends.sql` were authored by hand
and registered in `meta/_journal.json` manually because some of them
(notably adding a composite PRIMARY KEY to an existing table, and the
schema dedup step in `0002`) need imperative SQL that `drizzle-kit`
won't emit cleanly.

`0006_worried_surge.sql` is a deliberate no-op baseline reconciliation
migration. It exists solely to anchor `meta/0006_snapshot.json` after
the hand-rolled migrations had already been applied.

The schema in `src/db/schema.ts` is the source of truth for the *current*
state — all five hand-rolled migrations have been applied to the live
schema there. The `meta/` snapshot JSON files now cover both the original
`0000_messy_proteus` baseline and the reconciled `0006` baseline.

## What this means in practice

- **Applying migrations is fine** — `npm run db:migrate` walks the
  journal entries in order and runs the `.sql` files.

- **Generating new migrations** should now diff against
  `meta/0006_snapshot.json`, so future `npm run db:generate` runs stay
  focused on real schema changes instead of re-emitting the auth/daily/
  friends tables as a fake full rebuild.

## Migration list

| #    | Name                  | Why                                                                  |
| ---- | --------------------- | -------------------------------------------------------------------- |
| 0000 | `messy_proteus`       | Initial `cards` + `user_progress` tables (drizzle-generated).        |
| 0001 | `add_auth`            | Auth.js tables + `user_progress(user_id, answered_at)` index.        |
| 0002 | `user_progress_pk`    | Composite PK on `user_progress` after dedup. Stops race-condition duplicates. |
| 0003 | `track_correct`       | Adds `user_progress.correct` + partial index on wrong answers. Unlocks spaced repetition. |
| 0004 | `daily_results`       | `daily_results(user_id, date)` table for Daily Challenge.            |
| 0005 | `friends`             | `users.friend_code` + `friends(user_id, friend_user_id)` for social features. |
| 0006 | `worried_surge`       | No-op baseline reconciliation so Drizzle snapshots align with the current schema. |
